Nesta aula, aprendemos como criar views no PostgreSQL para transformar 
tabelas virtuais em objetos persistentes no banco de dados.

Também foi explicado que uma boa prática é adicionar o prefixo 
vw_ aos nomes das views para facilitar a identificação.

Além disso, foi mostrado como consultar uma view como se fosse uma tabela 
comum e como combinar views com outras tabelas ou aplicar filtros.

Comandos SQL utilizados na aula:

CREATE VIEW vw_cursos_por_categoria AS SELECT categoria.nome AS categoria,
			COUNT (curso.id) AS numero_cursos
		FROM categoria
		JOIN curso ON curso.categoria_id = categoria.id
GROUP BY categoria;

SELECT * 
FROM vw_cursos_por_categoria;

CREATE VIEW vw_cursos_tecnologia AS 
	SELECT curso.nome AS nome_curso, cat.nome AS nome_categoria
	FROM curso
	LEFT JOIN categoria cat ON curso.categoria_id = cat.id
	WHERE cat.id = 1;

SELECT *
FROM vw_cursos_tecnologia;
