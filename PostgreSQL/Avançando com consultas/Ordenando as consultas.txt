Nesta aula, aprendemos como ordenar os resultados de uma consulta no PostgreSQL utilizando a cláusula ORDER BY.

Principais pontos abordados:

Ordenação básica:
  - Utilização de ORDER BY nome_do_campo para ordenar os resultados em ordem crescente (padrão).
  - Utilização de ORDER BY nome_do_campo DESC para ordenar os resultados em ordem decrescente.

Ordenação por múltiplos campos:
  - É possível ordenar por mais de um campo, definindo a prioridade na ordem em que os campos são listados (ex: ORDER BY nome, matricula DESC).

Ordenação por posição:
  - É possível ordenar pelo número da coluna na tabela (ex: ORDER BY 4 para ordenar pelo quarto campo).

Desambiguação de campos:
  - Quando um campo existe em mais de uma tabela (após um JOIN, por exemplo), é necessário especificar a tabela para evitar ambiguidade (ex: ORDER BY aluno.nome).

Comandos SQL utilizados nessa aula:

DROP TABLE funcionarios;
DROP TABLE professores;

CREATE TABLE funcionarios (
	id SERIAL PRIMARY KEY,
	matricula VARCHAR(10),
	nome VARCHAR(255),
	sobrenome VARCHAR(255)
);
CREATE TABLE professores (
	id SERIAL PRIMARY KEY,
	matricula VARCHAR(10),
	nome VARCHAR(255),
	sobrenome VARCHAR(255)
);

SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
